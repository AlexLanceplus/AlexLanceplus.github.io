动态库与静态库

函数库的意义是避免重复重写一些已经很完善的函数，比如printf等，函数库分为两种，静态库与动态库，静态库一般为libxxx.a，动态库则一般为libxxx.so,也有可能是libxxx.so.major.mininor，major表示主版本号，mininor则表示次版本号

动态库与静态库的链接方式也不相同，静态库的链接方式每当一个源文件调用该函数库的时候便将整个函数库链接进来，形成一个完整的完整的可执行文件，该文件可以脱离函数库独立存在，但是缺点是内存消耗大，耗费空间，而动态库则是动态链接，每当源文件调用函数库的时候，便会将函数所需要的部分链接，当其他源文件调用相同函数的时候，则会通过地址映射的方式映射函数库,但是缺点也很明显，生成的可执行文件的运行离不开函数库

Linux里的库一般储存在/user/lib /lib

在gcc编译环境下，静态库与动态库同名时，系统优先使用动态库

静态库的生成

1 将 xx.c 文件生成 xx.o 文件 gcc -c xx.c

2 将 xx.o 文件打包制作成 libxx.a 文件 ar rc libxx.a xx.o

3 将 libxx.a 放到系统库，或指定链节路径下使用

动态库的生成

1 将 xx.c 文件生成 xx.o 文件 gcc -c xx.c

2 将 xx.o 打包生成 libxx.so gcc -shared -fPIC -o libxx.so xx.o

3 将 libxx.a 放到系统库，或指定链节路径下使用

链接路径：

链接时找不到某一个函数，出现" xxxxx undefined”，应该就是链接时找不到相应的 动态库，连接时动态库的路径如下：

1 默认路径是/lib,/usr/lib/, /usr/local/lib

2 -L: 指定连接时动态库的路径

3 LD_LIBRARY_PATH: 指定连接路径

运行路径

运行应用程序时出现 "error while loading shared libraries” ，则是加载时找不到相 应的动态库

1 默认的动态库搜索路径/lib；

2 默认的动态库搜索路径/usr/lib

3 环境变量 LD_LIBRARY_PATH 指定的动态库搜索路径；

4 修改配置文件/etc/ld.so.conf 中指定的动态库搜索路径来改变,然后运行 ldconfig 更新一下，/etc/ld.so.cache

sqlite3

SQLite 是一个开源的嵌入式关系数据库，实现自包容、零配置、支持事务的 SQL 数据 库引擎。 其特点是高度便携、使用方便、结构紧凑、高效、可靠，数据库的本质是文件，可以类比Excel文件，数据库文件以.db结尾，与Excel类似，数据库由许多表（table）组成，表又包括列与行

1.sqlite3.exe [dbname] 打开 sqlite3 的交互模式，[并创建数据库]

2 .open dbname 若未创建数据，可以创建数据库，若己创建则打开

3 .databases 显示当前打开的数据库文件

4 .exit 退出交互模式

5 .help,列出命令的提示信息，可供于查阅

1.open mydb 打开要创建表的那个数据库。

2 create table (field1 type1, field2 type2,…); 建表语句

3 .tables 查看当前数据库下所有的表

4 .schema [tbname] 查看表结构(主要看列信息)

5 drop table ; 销毁表的语句

1.tables 插入数据之前先确定，所要插入表的存在

2 insert into values (value1, value2,…);向表中添加新记录 insert into (key1, key2) values(value1, value2); 值和名字对应即可

3 select filed1,field2 … from ;查询表中某些字段的记录

4 select * from ;查询表中所有字段的记录

5 .header on 显示字段名称

6 .mode column 以列模式显示字段的记录。默认是 list 模

order by field desc|asc 此子句，可以用于表达排序，desc 表示降序，asc 表示 升序

1 update set ,…where ; 更新表中记录。若没有 where 子句，则会全部修改。

2 delete from where ; 若没有 where 子句，则 会删全表内容，但不同于 drop
